# Time to Value: Optimizing User Onboarding

We fixed our onboarding problem by removing a decision—and optimizing for time to value.

Early versions of WatchWithMe had a familiar flow:
- Install
- **Sign up / log in**
- Create a room
- Start watching

On paper, it looked fine. In practice, it slowed down **time to value**.

People weren't dropping off because the product didn't work. They were dropping off because we were asking them to **decide too early**, before they experienced anything meaningful.

## The Realization

WatchWithMe is not a form-driven product. It's an experience product.

Users don't come to "create an account". They come to watch something together.

So we reframed our onboarding goal around a single question:

> **How fast can a new user reach their first moment of value?**

For us, that meant one clear metric:

> **A user should be able to create their first room within the first 10 minutes of installation.**

Everything else became secondary.

## What We Changed

We made **anonymous login the default**.

- Users can start using the product immediately
- No forced Google login upfront
- No decisions before value
- Authentication becomes an **upgrade**, not a gate

Once users experience the product and create a room, they can choose to upgrade to Google login for persistence, sharing, or return access.

## The Principle

**Reduce friction. Improve time to value. Earn commitment later.**

## The Outcome

- Faster time to value
- Less hesitation and confusion during onboarding
- Signup completion increased by ~3× after switching to anonymous-first access

Auth stopped feeling like friction. It became an invitation.

## When to Delay Authentication

### Good Candidates for Anonymous-First
- Experience products (video, games, content)
- Low-stakes tools (utilities, productivity)
- Products with clear immediate value

### Bad Candidates for Anonymous-First
- Financial services (compliance requires identity)
- Healthcare (legal requirements)
- Enterprise B2B (audit trails matter)

## The Framework

Ask: **"What is the first moment of value?"**

Then work backwards:
1. What does the user need to do to get there?
2. What can we delay without hurting that experience?
3. What decision can we remove entirely?

Usually, you can delay:
- Account creation
- Profile completion
- Settings configuration
- Payment collection

## The Meta-Lesson

**Users don't owe you their information before they've experienced your value.**

Make it easy to start. Make the value undeniable. Then ask for commitment.

That's how you build products people love—not tolerate.
